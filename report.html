Resumen del Sistema

El cargador de currículums de Konexa es una herramienta desktop desarrollada en Python para estandarizar perfiles técnicos de postulantes. Convierte CVs sueltos en formato PDF o Word (DOCX) en registros JSON estructurados y luego exporta PDFs maquetados de forma homogénea con la identidad de la marca.

Propósito de los Archivos en el Repositorio

El repositorio conserva algunos archivos históricos que difieren en el comportamiento lógico del parser:

  • 🐍
    CVParser.py: Código definitivo de producción. Soporta carga individual con editor GUI, procesamiento por cola múltiple y empaquetamiento macOS (.spec).
  • 📄
    cv_parser.py: Script de pruebas heredado. Utiliza la llamada clásica de completions (chat.completions.create) con el modelo gpt-4-1106-preview. No contiene lógica de colas seguras.

Flujo de Ejecución del Programa

El script de producción ejecuta hilos en paralelo para evitar congelar la interfaz gráfica de Tkinter durante los tiempos de respuesta de la API de OpenAI o la conversión de documentos.

⬇

1. Flujo de Parsing Individual

  • A
    Selección y Subida: Se abre el selector nativo. Los archivos Word se convierten inmediatamente a PDF. El PDF se sube a los servidores de OpenAI mediante client.files.create.
  • B
    Extracción por IA: Se envía el identificador del archivo al modelo indicándole las reglas del JSON. La respuesta se parsea y se guarda localmente como ~/Documents/parser/<Nombre Apellido>.json.
  • C
    Edición y Compilación: Se activa edit_cv(), abriendo la ventana interactiva. Al guardar, se actualiza el JSON local y WeasyPrint compila el PDF de salida.

2. Flujo de Parsing Múltiple (Cola)

A través del botón "Parse Multiple CVs", la aplicación implementa una ejecución por cola:

  • 1
    Llamada Asíncrona: Para cada archivo seleccionado, se instancia un objeto CVParser y se invoca run_parser_thread().
  • 2
    Generación Directa: A diferencia de la carga individual, la cola múltiple utiliza la función callback on_parse_complete para compilar el PDF de inmediato (generate_pdf()) sin abrir la ventana del editor.

Reglas de Extracción de Datos e Inteligencia Artificial

El prompt de extracción está hardcodeado en la propiedad self.instructions del constructor de la clase CVParser en [CVParser.py](file:///Users/nicolasherrera/Library/CloudStorage/OneDrive-EVERTEC/Python/PythonTestingLab/CV_parser_APP/CVParser.py#L679). Obliga al modelo a comportarse como un estructurador JSON estricto.

Directrices Forzadas en el Prompt

  • 🔹
    Rut y Ubicación: El campo rut debe extraerse del documento de identidad o pasaporte. El campo ubicacion debe normalizarse para contener únicamente la comuna chilena de residencia del candidato.
  • 🔹
    Resumen Profesional con HTML: Si el candidato no tiene resumen, la IA lo genera. Debe comenzar con la profesión y años de experiencia, e incluir las principales herramientas y tecnologías destacadas utilizando negritas HTML (<strong>tecnología</strong>).
  • 🔹
    Cronología de Experiencias: Ordena obligatoriamente el array de experiencias desde la más reciente a la más antigua. Además, separa los párrafos internos de las tareas usando saltos de línea <br>.
  • 🔹
    Diccionario de Idiomas: Asume español nativo a menos que el CV lo contradiga. Si no se detalla el nivel de inglés, deja la cadena vacía "" en el diccionario estructurado.

Plantillas HTML y CSS: Análisis de Variantes

La generación de PDF se basa en la interpolación de datos sobre plantillas HTML/CSS compiladas con Jinja2. Existen tres implementaciones principales que contienen variaciones sutiles y errores sintácticos de CSS/HTML que deben registrarse.

Plantilla Hardcoded en CVParser.py

Es el fallback que escribe el script si no encuentra plantillas en el sistema de carpetas. Tiene las siguientes propiedades distintivas:

  • Ausencia de fuentes web: No declara imports para fuentes, lo que provoca que WeasyPrint recurra a fuentes locales (como Ubuntu) si están preinstaladas en la máquina.
  • Declaración CSS: body { font-family: Ubuntu, Open-Sans; } (Sin comillas).
  • Título de Sección con tilde: Renderiza la etiqueta como <h2>Educación</h2>.
  • Vulnerabilidad Estructural: Contiene un bloque de cabecera inválido nested dentro del div `.title`: 
    <div class="title">
    <head>
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <h1>{{ hunting_name }}</h1>
    </head>
    <h2 style="gap:10px">{{ bio.profesion }}</h2>
</div>

Hunting.html (Plantilla en Disco)

Es la plantilla activa copiada por defecto. Difiere sutilmente de la versión hardcoded:

  • Importación de fuentes web: Incluye el bloque @import url('https://fonts.googleapis.com/css2?family=Ubuntu...'); para forzar la descarga en el render de WeasyPrint.
  • Declaración CSS: body { font-family: "Ubuntu", Open-Sans; } (Con comillas en Ubuntu).
  • Título de Sección plano: La sección académica se define como <h2>Educacion</h2> (Sin tilde).
  • Posee la misma estructura de metadatos inválida anidada en el body.

Second Template.html (Plantilla Alternativa)

Representa una reestructuración de diseño completamente distinta:

  • Layout a una columna: No utiliza el grid/tabla de dos columnas. Muestra cabecera, perfil, stack tecnológico horizontal, experiencias, formación e idiomas en orden descendente continuo.
  • Asset de Logo SVG: Contiene una etiqueta de imagen con el logo de Konexa codificado directamente como SVG en base64 en lugar del asset PNG.
  • Propiedades de Página: Añade un color de fondo plano a nivel A4: @page { size: A4; margin: 18mm; background-color: #f8fafa; }.

Vulnerabilidades Lógicas y Bugs Identificados

Durante la revisión profunda del código fuente, se identificaron varios fallos críticos a nivel de diseño de arquitectura y control de excepciones en [CVParser.py](file:///Users/nicolasherrera/Library/CloudStorage/OneDrive-EVERTEC/Python/PythonTestingLab/CV_parser_APP/CVParser.py).

1. Bug del Index Out of Range en PDF Filename

En la función de compilación final de PDF generate_pdf (Línea 666): 

pdf_filename = f"{CV['bio']['nombres'].split()[0]} {CV['bio']['apellidos'].split()[0]}_CV.pdf"

El Problema: Si el candidato no tiene nombres o apellidos estructurados en el JSON (o si el usuario los borra en la ventana del editor antes de guardar), la función .split() devolverá una lista vacía. Acceder a la posición [0] levantará un error de tipo IndexError que detendrá la ejecución del hilo y abortará la creación del PDF, sin mostrar una alerta coherente al usuario.

2. Dependencia Oculta de MS Word en docx2pdf

Para procesar archivos con formato Microsoft Word: 

from docx2pdf import convert
...
if file_path.endswith(".docx"):
    convert(file_path, file_path[:-5]+".pdf")

El Problema: La librería docx2pdf no es un conversor autónomo. Utiliza enlaces COM (Windows) o AppleScript (macOS) para invocar una instancia en background de Microsoft Word instalada en la máquina. Si el usuario que ejecuta la app no cuenta con una licencia activa de Office, el script levantará un error fatal impidiendo el procesamiento del archivo `.docx`.

3. Colisión de Foco UI en Ejecuciones Concurrentes

Al lanzar tareas masivas a través de parse_multiple_cvs, el programa instancia y ejecuta múltiples hilos simultáneos: 

for file_path in files:
    parser = CVParser(...)
    parser.run_parser_thread()

El Problema: Cada instancia del parser ejecuta el método show_loading, el cual realiza llamadas a la ventana principal usando grab_set() para bloquear eventos fuera de sí: 

self.loading_window.grab_set()

Dado que los hilos se ejecutan concurrentemente y compiten por levantar ventanas secundarias (Toplevel), esto causa un parpadeo constante en macOS/Windows y bloquea el flujo principal del hilo principal (GUI thread), provocando cuelgues temporales de la interfaz.

4. Conexión Insegura de API Keys

Las API Keys de OpenAI se guardan localmente en texto plano en la ruta: ~/Documents/parser/api_keys.json.

Cualquier otra aplicación local o usuario del sistema con permisos básicos de lectura puede leer o exfiltrar las claves. No existe encripción simétrica ni resguardo de credenciales nativas del sistema operativo (Keychain / Windows Credential Manager).
